# Glossary

| Term | Definition |
|------|------------|
| **Artifact** | A deliverable that represents a piece of work (e.g., code, documentation, test). |
| **Evidence** | Documentation or records proving an artifact meets quality or compliance standards. |
| **Quality Gate** | An automated check that must pass before an artifact advances in the pipeline. |
| **Lifecycle Stage** | A defined point in the progression of an artifact: Creation → Review → Validation → Deployment. |
| **ADR (Architectural Decision Record)** | A formal record of a design decision, its context, alternatives, and rationale. |
| **CI/CD** | Continuous Integration/Continuous Delivery – automated pipelines that build, test, and deploy artifacts. |
| **Version Control System (VCS)** | Tool for managing changes to source code and related artifacts (e.g., Git). |

This glossary supports consistent terminology across all documentation and communication within AIEF.
